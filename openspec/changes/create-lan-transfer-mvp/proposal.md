## Why

当前用户在多台电脑之间传文件时常依赖微信、网盘或聊天工具中转，流程长且不符合局域网内设备直连的使用直觉。本变更基于 `docs/PRD-v0.1.md` 和 `UI/` 参考图，定义局域网文件传输工具 v0.1 MVP 的产品与技术规格，为后续实现提供清晰边界。

## What Changes

- 新增桌面端局域网文件传输 MVP：两台同一局域网桌面设备启动应用后，可通过手动输入接收方地址建立连接并发送文件。
- 新增本机信息与本地接收服务能力：展示设备名、推荐局域网访问地址、候选 IP、服务状态，并处理默认端口冲突。
- 新增手动连接目标设备能力：支持 IPv4 地址、端口和 HTTP URL 输入规范化，连接测试，目标设备信息展示，以及最近连接设备记录。
- 新增文件选择与待发送列表能力：支持点击选择和拖拽添加一个或多个文件，展示数量、大小、总大小，并允许发送前移除。
- 新增 HTTP 文件传输协议：发送请求、接收确认、状态轮询、顺序上传多文件、传输进度与失败状态。
- 新增接收确认与文件保存能力：接收方确认或拒绝传输，处理重名覆盖确认，将文件保存到默认或用户配置目录，并支持打开保存目录。
- 新增运行期传输任务列表：展示发送/接收方向、对方设备、文件、进度、状态和基础操作。
- 明确 v0.1 非目标：不包含公网传输、账号体系、云端中转、移动端、自动设备发现、二维码连接、断点续传、文件夹传输、完整端到端加密、多人群组和剪贴板同步。
- 本步骤仅创建 OpenSpec 变更文档，不生成应用代码。

## Capabilities

### New Capabilities

- `local-device-service`: 定义本机设备信息、局域网地址展示、IP 候选选择、本地 HTTP 服务监听和端口冲突处理。
- `manual-target-connection`: 定义手动输入目标地址、地址规范化、连接测试、协议兼容判断和最近连接设备记录。
- `file-selection-queue`: 定义文件选择、拖拽添加、待发送列表、文件大小展示、发送前校验和重名处理前置规则。
- `lan-transfer-protocol`: 定义 v0.1 HTTP 传输接口、发送请求、状态轮询、顺序上传多文件、进度事件和传输错误。
- `receive-and-save-files`: 定义接收确认、单任务接收限制、拒绝规则、保存目录、目录可用性检查、重名覆盖和打开保存目录。
- `transfer-task-state`: 定义发送和接收任务的运行期展示、方向、状态、进度、失败、拒绝和非持久化规则。
- `main-window-ui`: 定义 v0.1 单主页面结构、顶部状态、目标连接、文件发送、接收请求、任务列表和 UI 参考图约束。

### Modified Capabilities

- 无。当前 `openspec/specs/` 下尚无既有能力规格，本变更全部为新增能力。

## Impact

- 影响桌面应用主流程：React + TypeScript + Tailwind CSS 主窗口、Zustand 状态、Tauri command/event、本地配置和文件选择交互。
- 影响 Rust 本地能力：Tauri 2 集成、设备信息读取、网络接口枚举、保存目录管理、打开目录、本地配置读写、文件流式读取和写入。
- 影响本地 HTTP 服务：基于 axum + tokio 启动 `0.0.0.0:实际端口` 服务，并提供 `/api/device`、`/api/transfer/request`、`/api/transfer/status/:transferId`、`/api/transfer/upload` 等接口。
- 影响局域网通信约定：v0.1 仅支持 HTTP、IPv4 手动连接和协议版本 `0.1`，不引入 WebSocket/SSE。
- 影响用户可见体验：主页面遵循 `UI/design.png` 的三栏工作台布局，接收确认遵循 `UI/confirm.png` 的弹窗式确认体验。
