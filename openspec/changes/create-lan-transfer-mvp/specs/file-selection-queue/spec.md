## ADDED Requirements

### Requirement: 添加待发送文件

系统 MUST 支持用户通过文件选择器或拖拽方式添加一个或多个待发送文件。

#### Scenario: 通过选择器添加文件
- **WHEN** 用户点击选择文件并选择一个或多个文件
- **THEN** 系统 MUST 将所选文件加入待发送列表

#### Scenario: 通过拖拽添加文件
- **WHEN** 用户将一个或多个文件拖入发送区域
- **THEN** 系统 MUST 将拖入文件加入待发送列表

#### Scenario: 不支持文件夹传输
- **WHEN** 用户选择或拖入文件夹
- **THEN** 系统 MUST 在 v0.1 中拒绝将文件夹作为待发送项

### Requirement: 展示待发送列表信息

系统 MUST 展示待发送文件的文件名、单文件大小、文件数量和总大小。

#### Scenario: 展示多文件汇总
- **WHEN** 待发送列表包含多个文件
- **THEN** 系统 MUST 展示已选择文件数量和全部文件总大小

#### Scenario: 展示单文件条目
- **WHEN** 待发送列表包含文件
- **THEN** 系统 MUST 对每个文件展示文件名和文件大小

### Requirement: 管理待发送列表

系统 MUST 允许用户在发送前移除待发送文件，并根据列表状态控制发送按钮可用性。

#### Scenario: 移除待发送文件
- **WHEN** 用户点击某个待发送文件的移除操作
- **THEN** 系统 MUST 从待发送列表删除该文件

#### Scenario: 空列表禁用发送
- **WHEN** 待发送列表为空
- **THEN** 系统 MUST 禁用发送按钮

#### Scenario: 未连接目标禁用发送
- **WHEN** 当前目标设备未完成成功连接测试
- **THEN** 系统 MUST 禁用发送按钮或阻止发送动作

### Requirement: 发送前校验文件可读性

系统 MUST 在发送开始前校验待发送文件仍存在且可读取。

#### Scenario: 文件已删除
- **WHEN** 用户点击发送但某个待发送文件已不存在
- **THEN** 系统 MUST 阻止本次发送并提示文件不可用

#### Scenario: 文件不可读取
- **WHEN** 用户点击发送但某个待发送文件无读取权限
- **THEN** 系统 MUST 阻止本次发送并提示文件不可读取

### Requirement: 不设置文件大小限制

系统 MUST 在 v0.1 中不主动限制单文件大小或待发送总大小。

#### Scenario: 选择大文件
- **WHEN** 用户选择任意大小的单个文件
- **THEN** 系统 MUST 允许加入待发送列表

#### Scenario: 选择多个文件
- **WHEN** 用户一次选择多个文件
- **THEN** 系统 MUST 允许加入待发送列表并计算总大小
